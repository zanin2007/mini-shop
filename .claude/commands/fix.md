아래 코드 리뷰 내용을 바탕으로 코드를 수정해줘.

$ARGUMENTS

수정 규칙:
1. **심각도 순서대로 수정**: Critical → High → Medium → Low
2. **수정 전에 해당 파일을 먼저 읽고** 현재 코드 상태를 확인해
3. **각 수정마다 설명**: 어떤 파일의 어떤 부분을 왜 수정했는지 간결하게 알려줘
4. **사이드 이펙트 체크**: 수정한 코드가 다른 곳에 영향을 주는지 확인해
5. **빌드 확인**: 프론트엔드 수정이 포함되면 `npm run build`로 빌드 에러 없는지 확인해

수정 완료 후 변경 파일 목록과 요약을 표로 정리해줘.
