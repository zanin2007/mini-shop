현재 git diff에 있는 변경사항을 코드 리뷰해줘.

다음 관점으로 검토해줘:
1. **보안**: SQL injection, XSS, 권한 우회 취약점
2. **동시성**: Race condition, 이중 처리, 재고 초과 가능성
3. **엣지 케이스**: null, 0, 음수, 빈 배열 등 경계값
4. **성능**: N+1 쿼리, 불필요한 반복, 인덱스 누락
5. **타입 안전성**: any 사용, 타입 누락 (프론트엔드)

각 이슈를 심각도(Critical/High/Medium/Low)로 분류하고, 수정안도 함께 제시해줘.
변경사항이 없으면 "변경사항이 없습니다"라고 알려줘.
